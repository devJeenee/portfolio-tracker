# 프로젝트 강점 분석

현재 프로젝트에서 잘 구현된 패턴과 기술적 강점을 찾습니다.

## 실행할 작업

`portfolio_insight` MCP 도구를 사용하세요:

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `sections`: ["strengths"]

## 출력 형식

강점을 카테고리별로 정리하고, 각 강점에 대해:
- 무엇이 잘 되어있는지
- 포트폴리오에서 어떻게 어필할 수 있는지
- 관련 키워드

를 한국어로 설명해주세요.

## 에러 대응

- **path에 pom.xml / build.gradle 없음**: "이 경로에서 Spring Boot 프로젝트를 찾을 수 없습니다. Spring Boot 프로젝트의 루트 디렉토리를 지정해주세요. (예: `/strengths ./my-spring-app`)" 라고 안내
- **Git 저장소가 아닌 경우**: "Git 저장소가 아니어서 일부 분석이 제한됩니다." 라고 안내하고, 코드 기반 강점만 분석
- **강점이 0개인 경우**: 다음을 분석하여 안내하세요:
  - "현재 감지된 강점이 없습니다. 가능한 원인:"
  - "1) 프로젝트가 아직 초기 단계 — 기본 구조부터 잡아가면 강점이 생깁니다"
  - "2) 분석 범위 밖의 패턴 사용 — 사용 중인 기술을 알려주시면 수동으로 분석해드립니다"
  - "`/opportunities`로 개선 포인트를 먼저 확인하고 개선하면 강점이 늘어납니다."
- **분석 결과가 비어있는 경우**: "분석 결과가 비어있습니다. 프로젝트 루트 경로가 맞는지 확인해주세요." 라고 안내

## 후속 제안

- "각 강점에서 **이력서/면접 키워드**를 추출하고 싶다면, 번호를 알려주세요. 면접용 키워드와 예상 질문을 만들어드립니다."
- "부족한 영역을 보완하고 싶다면 `/opportunities`로 개선 기회를 확인하세요."
- "전체 포트폴리오 가치가 궁금하다면 `/portfolio`로 종합 분석을 받아보세요."
