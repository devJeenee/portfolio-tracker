# 포트폴리오 종합 분석

프로젝트의 포트폴리오 가치를 멀티 스텝으로 종합 분석합니다.

## 실행할 작업

다음 단계를 순서대로 진행하세요:

### Step 1: 프로젝트 구조 파악

`analyze_project` MCP 도구로 프로젝트의 구조와 기술 스택을 먼저 파악합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)

### Step 2: 포트폴리오 종합 분석

`portfolio_insight` MCP 도구로 전체 분석을 수행합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `sections`: ["all"]

### Step 3: 심각도 높은 이슈 상세 분석 (조건부)

Step 2 결과에서 심각도(severity) "high" 이슈가 **3개 이상**이면, `analyze_code` MCP 도구로 해당 영역의 상세 코드 분석을 추가 수행합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)

3개 미만이면 이 단계를 건너뜁니다.

### Step 4: 종합 리포트 작성

모든 분석 결과를 종합하여 다음 형식으로 한국어 리포트를 작성하세요:

1. **한줄 요약**: 프로젝트의 포트폴리오 가치를 한 문장으로
2. **어필 포인트 TOP 5**: 면접/이력서에서 바로 쓸 수 있는 핵심 강점
3. **성숙도 점수**: 종합 점수 + 등급(S/A/B/C/D/F) + 카테고리별 점수
4. **주요 경험**: impact 순서대로 상위 경험들 (STAR 형식 힌트 포함)
5. **즉시 개선 TOP 3**: 가장 효과적인 개선 3가지 (구체적 방법 포함)
6. **이력서용 초안**: 이 프로젝트를 이력서에 쓸 때의 2-3줄 서술문 초안

## 에러 대응

- **path에 pom.xml / build.gradle 없음**: "이 경로에서 Spring Boot 프로젝트를 찾을 수 없습니다. Spring Boot 프로젝트의 루트 디렉토리를 지정해주세요. (예: `/portfolio ./my-spring-app`)" 라고 안내
- **Git 저장소가 아닌 경우**: "Git 저장소가 아니어서 경험 분석이 제한됩니다. `git init` 후 커밋 히스토리를 쌓으면 더 풍부한 분석이 가능합니다." 라고 안내하고, Git 없이 가능한 분석만 진행
- **분석 결과가 비어있는 경우**: "분석 결과가 비어있습니다. 가능한 원인: 1) 소스 코드가 아직 없음 2) 지원하지 않는 프로젝트 구조 3) 경로가 잘못됨. 프로젝트 루트 경로를 확인해주세요." 라고 안내

## 후속 제안

분석 결과의 성숙도 등급에 따라 맞춤 제안을 제공하세요:

- **S/A 등급**: "이미 높은 완성도입니다! `/strengths`로 강점을 더 구체화하여 면접 준비에 활용해보세요."
- **B/C 등급**: "개선 여지가 있습니다. `/opportunities`로 가장 효과적인 개선 포인트를 확인하고, `/maturity`로 세부 점수를 분석해보세요."
- **D/F 등급**: "기초 구조부터 개선이 필요합니다. `/analyze`로 코드 품질 이슈를 먼저 해결하고, `/opportunities`로 단계별 개선을 시작하세요."
- 공통: "`/experiences`로 Git 히스토리 기반 경험을 더 발굴할 수 있습니다."
