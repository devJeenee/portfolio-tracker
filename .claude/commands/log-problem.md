# 문제 기록

개발 중 만난 문제를 포트폴리오 추적용으로 기록합니다.

## 실행할 작업

`log_problem` MCP 도구를 사용하세요.

$ARGUMENTS 를 파싱하여 문제 내용을 전달해주세요.

### 입력 가이드

$ARGUMENTS가 **비어있거나 너무 짧은 경우** (10자 미만), 대화형으로 다음을 순서대로 질문하세요:

1. **문제 설명**: "어떤 문제를 만났나요? (예: 'JPA N+1 쿼리 성능 이슈')"
2. **맥락** (선택): "어떤 상황에서 발생했나요? (예: '대량 데이터 조회 시'). 건너뛰려면 엔터"
3. **태그** (선택): "관련 키워드가 있나요? (예: 'performance, jpa'). 건너뛰려면 엔터"

$ARGUMENTS가 **충분한 경우**, 내용을 분석하여 자동으로 파라미터를 구성하세요:
- `description`: 핵심 문제 설명
- `context`: 맥락/상황 (추출 가능한 경우)
- `tags`: 관련 키워드 (자동 추출)
- `path`: 현재 디렉토리

## 에러 대응

- **기록 실패**: "문제 기록에 실패했습니다. MCP 서버가 실행 중인지 확인해주세요." 라고 안내
- **분석 결과가 비어있는 경우**: "문제를 기록할 수 없습니다. 설명이 너무 짧거나 비어있을 수 있습니다. 문제를 좀 더 구체적으로 설명해주세요." 라고 안내

## 후속 제안

문제가 성공적으로 기록되면 다음을 안내하세요:

- "문제가 기록되었습니다! (ID: {기록된 ID})"
- "이 문제를 해결한 후 `/resolve-problem {ID} 해결 방법`으로 해결 과정을 기록하면, 포트폴리오에서 **문제 해결 경험**으로 어필할 수 있습니다."
- "현재 프로젝트의 다른 개선 포인트가 궁금하다면 `/opportunities`로 확인해보세요."
