# 개선 기회 탐색

현재 코드에서 포트폴리오 가치가 높은 개선 포인트를 찾고, 구체적인 실행 가이드를 제공합니다.

## 실행할 작업

다음 단계를 순서대로 진행하세요:

### Step 1: 개선 기회 분석

`portfolio_insight` MCP 도구를 사용하세요:

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `sections`: ["opportunities"]

### Step 2: 상위 기회 상세 분석 (조건부)

Step 1 결과에서 portfolioValue가 높은 **상위 5개** 기회에 대해, `analyze_code` MCP 도구로 해당 파일/영역의 상세 코드 분석을 수행합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)

기회가 5개 미만이면 있는 만큼만 분석합니다.

### Step 3: 실행 가이드 작성

모든 개선 기회를 portfolioValue 순서로 정리하고, 각 항목에 대해 다음을 한국어로 작성하세요:

1. **타입 + 심각도 + 포트폴리오 가치 점수**
2. **파일 위치 + 현재 문제**
3. **구체적인 수정 방향**: 어떤 코드를 어떻게 바꿔야 하는지 (패턴/코드 예시 포함)
4. **예상 성숙도 점수 변화**: 이 개선을 하면 성숙도 점수가 대략 얼마나 오를지 추정
5. **포트폴리오 어필 포인트**: 개선 후 면접에서 어떻게 어필할 수 있는지

상위 10개까지만 보여주세요.

## 에러 대응

- **path에 pom.xml / build.gradle 없음**: "이 경로에서 Spring Boot 프로젝트를 찾을 수 없습니다. Spring Boot 프로젝트의 루트 디렉토리를 지정해주세요. (예: `/opportunities ./my-spring-app`)" 라고 안내
- **Git 저장소가 아닌 경우**: "Git 저장소가 아니어서 일부 분석이 제한됩니다. 코드 기반 분석만 진행합니다." 라고 안내하고, 가능한 분석만 수행
- **개선 기회가 0개**: "현재 감지된 개선 기회가 없습니다. 이미 높은 완성도이거나, 분석 범위 밖의 프로젝트 구조일 수 있습니다. `/strengths`로 현재 강점을 확인해보세요." 라고 안내
- **분석 결과가 비어있는 경우**: "분석 결과가 비어있습니다. 프로젝트 루트 경로가 맞는지 확인해주세요." 라고 안내

## 후속 제안

- "바로 개선을 시작하고 싶다면, 개선할 항목 번호를 알려주세요. 구체적인 코드 수정을 도와드립니다."
- "개선 결과를 리포트로 만들고 싶다면, 개선 후 `/portfolio`로 종합 분석을 다시 받아보세요."
- "현재 성숙도 점수와 등급이 궁금하다면 `/maturity`로 확인해보세요."
