# 코드 품질 분석

프로젝트의 코드 품질을 분석하고 개선점을 찾습니다.

## 실행할 작업

`analyze_code` MCP 도구를 사용하세요:

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)

## 출력 형식

분석 결과를 다음 순서로 한국어로 정리해주세요:

1. **감지된 이슈 요약**: 타입별 개수
2. **심각도 높은 이슈** (portfolioValue 8+): 파일, 문제, 해결방법
3. **성숙도 점수**: 종합 점수 + 등급
4. **우선 개선 항목**: 가장 효과적인 개선 3가지

## 에러 대응

- **path에 pom.xml / build.gradle 없음**: "이 경로에서 Spring Boot 프로젝트를 찾을 수 없습니다. 이 도구는 Java/Kotlin Spring Boot 프로젝트를 분석합니다. Spring Boot 프로젝트의 루트 디렉토리를 지정해주세요. (예: `/analyze ./my-spring-app`)" 라고 안내
- **지정된 파일/디렉토리가 없는 경우**: "해당 경로를 찾을 수 없습니다. 경로를 확인해주세요. 현재 디렉토리 기준으로 분석하려면 인자 없이 `/analyze`만 실행하세요." 라고 안내
- **Git 저장소가 아닌 경우**: "Git 저장소가 아니어서 히스토리 기반 분석이 제한됩니다. 코드 구조 기반 분석만 진행합니다." 라고 안내
- **분석 결과가 비어있는 경우**: "분석 결과가 비어있습니다. 소스 코드가 있는지, 경로가 올바른지 확인해주세요." 라고 안내

## 후속 제안

심각도별로 맞춤 안내를 제공하세요:

- **심각도 high 이슈가 있는 경우**: "심각도 높은 이슈가 있습니다. 바로 수정을 시작하고 싶다면 이슈 번호를 알려주세요. 구체적인 코드 수정을 도와드립니다."
- **심각도 medium 이슈만 있는 경우**: "치명적인 이슈는 없지만 개선 여지가 있습니다. `/opportunities`로 포트폴리오 가치 높은 개선 포인트를 확인해보세요."
- **이슈가 거의 없는 경우**: "코드 품질이 좋습니다! `/strengths`로 강점을 정리하여 면접에 활용해보세요."
- 공통: "`/portfolio`로 전체 포트폴리오 가치를 종합 분석할 수 있습니다."
