# 프로젝트 성숙도 평가

프로젝트의 전반적인 성숙도를 0-100점으로 평가하고, 일반적인 주니어 프로젝트와 비교 분석합니다.

## 실행할 작업

다음 단계를 순서대로 진행하세요:

### Step 1: 프로젝트 규모 파악

`analyze_project` MCP 도구로 프로젝트의 구조와 규모를 먼저 파악합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)

### Step 2: 성숙도 분석

`portfolio_insight` MCP 도구로 성숙도를 평가합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `sections`: ["maturity"]

### Step 3: 비교 분석 및 리포트 작성

분석 결과를 다음 형식으로 한국어 리포트를 작성하세요:

1. **종합 점수**: 0-100점 + 등급(S/A/B/C/D/F)
2. **카테고리별 점수표**: 각 카테고리의 점수와 감점 사유를 표 형식으로
3. **주니어 프로젝트 비교 코멘트**: 각 카테고리 점수를 "일반적인 주니어 프로젝트" 기준(아래 참고)과 비교하여 코멘트
   - 일반적인 주니어 프로젝트 기준:
     - 아키텍처: 40-50점 (기본 레이어드 아키텍처)
     - 보안: 20-30점 (기본 설정만)
     - 테스트: 10-20점 (테스트 거의 없음)
     - API 설계: 40-50점 (기본 CRUD)
     - 에러 핸들링: 20-30점 (기본적)
     - 문서화: 10-20점 (README 정도)
   - 기준 대비 높은 카테고리는 "강점"으로, 낮은 카테고리는 "보완 필요"로 표시
4. **개선 로드맵**: 우선순위별 개선 항목 (포트폴리오 가치 높은 순서)

## 에러 대응

- **path에 pom.xml / build.gradle 없음**: "이 경로에서 Spring Boot 프로젝트를 찾을 수 없습니다. Spring Boot 프로젝트의 루트 디렉토리를 지정해주세요. (예: `/maturity ./my-spring-app`)" 라고 안내
- **Git 저장소가 아닌 경우**: "Git 저장소가 아니어서 일부 카테고리 점수가 제한될 수 있습니다." 라고 안내하고 가능한 분석만 진행
- **분석 결과가 비어있는 경우**: "성숙도를 평가할 수 없습니다. 프로젝트에 소스 코드가 있는지, 경로가 올바른지 확인해주세요." 라고 안내

## 후속 제안

결과에 따라 다음을 안내하세요:

- **등급 올리기 가장 쉬운 3가지**: 카테고리별 점수를 분석하여 가장 적은 노력으로 점수를 올릴 수 있는 3가지를 구체적으로 제안 (예: "테스트 카테고리가 15점으로 가장 낮습니다. 핵심 서비스 3개에 단위 테스트만 추가해도 40점까지 올릴 수 있습니다.")
- "구체적인 개선 방법은 `/opportunities`로 확인하세요."
- "개선 후 다시 `/maturity`로 점수 변화를 확인해보세요."
- "전체 포트폴리오 가치가 궁금하다면 `/portfolio`로 종합 분석을 받아보세요."
