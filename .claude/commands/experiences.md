# Git 경험 발굴

Git 커밋 히스토리를 분석하여 포트폴리오에 쓸 수 있는 개선 경험을 찾습니다.

## 실행할 작업

다음 단계를 순서대로 진행하세요:

### Step 1: 경험 분석

`portfolio_insight` MCP 도구를 사용하세요:

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `sections`: ["experiences"]

### Step 2: 핵심 경험 심화 분석 (조건부)

Step 1 결과에서 impact가 **"high"**인 경험이 있으면, 해당 경험의 기간을 파악하여 `analyze_git` MCP 도구로 그 기간의 커밋을 추가 분석합니다.

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `since`: 해당 경험의 시작 시점 (커밋 날짜 기반 추정)
- `until`: 해당 경험의 종료 시점 (커밋 날짜 기반 추정)

high impact 경험이 없으면 이 단계를 건너뜁니다.

### Step 3: STAR 형식으로 가공

모든 경험을 impact(high/medium/low) 순서로 정리하고, 각 경험에 대해 다음을 한국어로 작성하세요:

1. **카테고리 + 제목**
2. **관련 커밋 수 + 파일**
3. **STAR 형식 서술문**:
   - **S(Situation)**: 당시 상황/배경
   - **T(Task)**: 해결해야 할 과제
   - **A(Action)**: 구체적으로 한 행동
   - **R(Result)**: 결과 및 개선 효과
4. **면접 키워드**: 이 경험에서 어필할 수 있는 기술 키워드

## 에러 대응

- **Git 저장소가 아닌 경우**: "Git 저장소가 아니어서 경험 분석을 수행할 수 없습니다. `git init` 후 커밋 히스토리를 쌓으면 개발 경험을 자동으로 발굴할 수 있습니다." 라고 안내
- **커밋이 10개 미만**: "커밋이 너무 적어 의미 있는 경험을 발굴하기 어렵습니다. 더 많은 커밋을 쌓은 후 다시 시도하거나, `/log-problem`으로 수동으로 경험을 기록해보세요." 라고 안내
- **분석 결과가 비어있는 경우**: "경험을 발굴할 수 없습니다. 커밋 메시지가 'fix', 'update' 등 단순한 경우 분석이 어려울 수 있습니다. 의미 있는 커밋 메시지 작성을 권장합니다." 라고 안내
- **path에 pom.xml / build.gradle 없음**: "이 경로에서 Spring Boot 프로젝트를 찾을 수 없습니다. Spring Boot 프로젝트의 루트 디렉토리를 지정해주세요." 라고 안내

## 후속 제안

- "STAR 서술문을 면접 답변으로 더 다듬고 싶다면, 각 경험 번호를 알려주세요. 구체적인 면접 답변 스크립트를 작성해드립니다."
- "관련 코드를 리뷰하고 싶다면, `/analyze`로 코드 품질을 확인해보세요."
- "전체 포트폴리오 가치가 궁금하다면 `/portfolio`로 종합 분석을 받아보세요."
