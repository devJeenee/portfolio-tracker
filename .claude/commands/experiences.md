# Git 경험 발굴

Git 커밋 히스토리를 분석하여 포트폴리오에 쓸 수 있는 개선 경험을 찾습니다.

## 실행할 작업

`portfolio_insight` MCP 도구를 사용하세요:

- `path`: $ARGUMENTS (비어있으면 현재 디렉토리)
- `sections`: ["experiences"]

## 출력 형식

발견된 경험을 impact(high/medium/low) 순서로 정리하고, 각 경험에 대해:
- 카테고리 + 제목
- 관련 커밋 수 + 파일
- **포트폴리오 서술문** (면접에서 바로 쓸 수 있도록)

를 한국어로 작성해주세요.
