# 문제 해결 기록

이전에 기록한 문제의 해결 방법을 기록합니다.

## 실행할 작업

`resolve_problem` MCP 도구를 사용하세요.

$ARGUMENTS 를 파싱하여 문제 ID와 해결 내용을 전달해주세요.

### 입력 가이드

$ARGUMENTS에서 다음을 파싱하세요:
- 첫 번째 토큰: 문제 **ID** (숫자 또는 식별자)
- 나머지: **해결 방법** 설명

예시: `/resolve-problem 3 FetchJoin과 BatchSize를 적용하여 N+1 문제 해결`

## 에러 대응

- **ID가 없는 경우** ($ARGUMENTS가 비어있거나 ID를 파싱할 수 없는 경우): "문제 ID를 지정해주세요. 사용법: `/resolve-problem {ID} {해결 방법}` (예: `/resolve-problem 3 FetchJoin 적용으로 해결`)" 라고 안내
- **해당 ID의 문제를 찾을 수 없는 경우**: "ID '{ID}'에 해당하는 문제를 찾을 수 없습니다. `/log-problem`으로 먼저 문제를 기록해주세요." 라고 안내
- **해결 방법 설명이 없는 경우**: "해결 방법을 함께 입력해주세요. 사용법: `/resolve-problem {ID} {해결 방법}` (예: `/resolve-problem 3 FetchJoin 적용으로 해결`)" 라고 안내
- **기록 실패**: "해결 기록에 실패했습니다. MCP 서버가 실행 중인지 확인해주세요." 라고 안내

## 후속 제안

해결이 성공적으로 기록되면 다음을 안내하세요:

- "문제 해결이 기록되었습니다! 이 경험은 포트폴리오에서 **문제 해결 능력**을 어필하는 좋은 소재입니다."
- "이 문제-해결 경험을 포함한 전체 포트폴리오 리포트를 생성하려면 `/portfolio`를 실행하세요."
- "더 많은 개발 스토리를 발굴하고 싶다면 `/git-story`로 Git 히스토리를 분석해보세요."
